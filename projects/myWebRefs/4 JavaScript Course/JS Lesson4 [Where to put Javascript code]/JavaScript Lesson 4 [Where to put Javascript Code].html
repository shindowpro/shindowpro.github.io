Where to put Javascript code

Main Styles to writing JavaScript:

=> There are (3) main Ways to write Javascript Codes:

  1. Internal Style

    2. => [Internal Style] is to write the main Javascript Code directly inside the main html Page between the Script Tag [ < script >   Javascript Codes   < /script > ]

    => [Internal Style] Can be written with in (2) different positions:

    1. [Internal Style] With in Head Tag [ < head >   < /head > ]

      2. This position will run First Javascript code before any other HTML Tag with in Body Tag, And this preferred to be just AFTER Link Tag [ < link   rel = "stylesheet"   href = "file.css" /> ] due to following reason:

      • To prevent any loading page stunning in Style and Link Tags which has imported CSS Files, due to probable error in Javascript code with in Script Tag in case if script Tag is before each Link Tag and Style Tag. And this happeninig becuase the Webpage browser regularly reading codes depending on it's First written priority on html page.

    2. [Internal Style] With in Body Tag [ < body >   < /body > ]

      3. In this position it is recomended to write the Javascript codes with in Script Tag just BEFORE the closing of Body Tag [ < /body > ] due to following reason:

      • Gurrantee a Proper loading of html Tags and Codes in case Script tag has any errors in it's Javascript codes.

    A/ [ Script Tag Internal Style ] (With in Head Tag) Example:

    Following codes is about using Script Tag and Javascript Codes in Optimal Position with in Head Tag:

    <!DOCTYPE html>
    <html>
       <head>
         <meta   charset = "utf-8"  />
         <link   rel = "styleSheet"  href = "File.css"  />
         <script> 
            alert("Hello Shadi");
            document.write("<p>This Paragraph from Javascript
            code</p>");
         </script>
       </head>
    </html>

    B/ [ Script Tag Internal Style ] (With in Body Tag) Example:

    Following codes is about using Script Tag and Javascript Codes in Optimal Position with in Body Tag:

    <!DOCTYPE html>
    <html>
       <head>
         <meta   charset = "utf-8"  />
         <link   rel = "styleSheet"  href = "File.css"  />
       </head>
       <body>
         <div> </div>
         <h3> </h3>
         <p> </p>
         <script> 
            alert("Hello Shadi");
            document.write("<p>This Paragraph from Javascript
            code</p>");
         </script>
       </body>
    </html>

  2. External Style

    3. => [Externl Style] is to import an External Javascript File from other Path, and allocate it to the current html File, with in [Script Tag] in (Src Attribute)

    => [Externl Style] is most likely to use when importing any External Code Source file wether it was JS or any other type, and Not using [Internal Style] except in case of certain Code for specific Page Element(s).

    < Script   src = "File.js" >   < /Script >

    => [External Style] Can be written with in (2) different positions:

    1. [External Style] With in Head Tag [< Head >   < /Head >]

      2. This position will run First Javascript code before any other HTML Tag with in Body Tag, And this preferred to be just AFTER Link Tag [ < link   rel = "stylesheet"   href = "file.css" /> ] due to following reasons:

      • To prevent any loading page stunning in Style and Link Tags which has imported CSS Files, due to probable error in Javascript code with in Script Tag in case if script Tag is before each Link Tag and Style Tag. And this happening becuase the Webpage browser regularly reading codes depending on it's First written priority on html page.
      • Also this position is better for page opening performance due to the faster saving External JS or any other file type in cache in comparison with if the codes were Internal Style.

    2. [External Style] With in Body Tag [ < body >  < /body >]

      3. In this position it is recomended to write the Javascript codes with in Script Tag just BEFORE the closing of Body Tag [ < /body > ] due to following reason:

      • Gurrantee a Proper loading of html Tags and Codes in case Script tag has any errors in it's Javascript codes.

    A/ [ Script Tag External Style ] (With in Head Tag) Example:

    Following codes is about using Script Tag and Javascript file Source in Optimal Position with in Head Tag:

    <!DOCTYPE html>
    <html>
       <head>
         <meta   charset = "utf-8"  />
         <link   rel = "styleSheet"  href = "File.css"  />
         <script  src="File.js" >  </script>
       </head>
    </html>

    B/ [ Script Tag Internal Style ] (With in Body Tag) Example:

    Following codes is about using Script Tag and Javascript Codes in Optimal Position with in Body Tag:

    <!DOCTYPE html>
    <html>
       <head>
         <meta   charset = "utf-8"  />
         <link   rel = "styleSheet"  href = "File.css"  />
       </head>
       <body>
         <div> </div>
         <h3> </h3>
         <p> </p>
         <script  src="File.js" >  </script>
       </body>
    </html>

  3. Inline Style

    4. => [Inline Style] is to Write Javascript code Directly with in html Tag with it's Attribute (function's name="")

    => [Inline Style] is rarely to use for Optimal Performance

    < div   onclick="alert('Hello This is Div');" >
    < div />

JS Style Priority and Notes:

=> The priority and commonly to use JS codes Style in HTML Documents even if it was an imported Javascript File(s) or direct codes, as following:

  1. [External Style]
  2. [Internal Style]
  3. [Inline Style]
