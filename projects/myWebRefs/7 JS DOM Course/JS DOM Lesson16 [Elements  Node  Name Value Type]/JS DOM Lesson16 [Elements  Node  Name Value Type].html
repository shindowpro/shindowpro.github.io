JS DOM [Element Node Name Value Type] Concept:

=> JS HTML DOM [Element Node Name Value Type]: These Methods are to have more Controling Tools of Element's child Nodes by controing it's properties {based on Object Concept} .

=> JS HTML DOM [Element Node Name Value Type]: These Methods can be used with all Types of Element's Child nodes (child Element , child Node) .

=> JS HTML DOM [Element Node Name Value Type]: These Methods can be used as get function or set Function .

=> JS HTML DOM [Element Node Name Value Type]: The following are Main properties-methtods of parent Element's child Nodes :

  • {child Node}'s Name [nodeName] => {ParElementVar.childNodes[x].nodeName;}

      • This Method is simmiler to the {tagName} of (child Node of element Type), but will get {null} value for other types of child Nodes

      This Method can produce different kind of Results upon the type of child Node as following :

      1. [#text] : if the childNode is type of (Empty space, Text, Comment)
      2. [tag Name] : if the childNode is type of (child Element)
  • {child Node}'s Value [nodeValue] => {ParElementVar.childNodes[x].nodeValue;}

      • This Method is different from the default value Methods, and require more determination of the specific child Node of the Selected child Node

      The result of this Direct method of the Public Child Node {if type of Child Node is (Text, Comment, empty Space)} with out specification is [#text]

      The result of this Direct method of the Public Child Node {if type of Child Node is (element)} with out specification is [null]

      This Direct Method can produce different Results if the Child Node is {Element Type}, with more child Node specification as follwoing :

      1. {ParElementVar.childNodes[x].firstElementChild.nodeValue;} => To Get the Value of the First Element child of the internal selected child Node
      2. {ParElementVar.childNodes[x].lastElementChild.nodeValue;} => To Get the Value of the Last Element child of the internal selected child Node
      3. {ParElementVar.childNodes[x].firstChild.nodeValue;} => To Get the Value of the First child Node (regardless of type of child Node) of the internal selected child Node
      4. {ParElementVar.childNodes[x].lastChild.nodeValue;} => To Get the Value of the Last child Node (regardless of type of child Node) of the internal selected child Node
      5. {ParElementVar.childNodes[x].childNodes[x].nodeValue;} => To Get the Value of the specific child Node (with certain index) (regardless of type of child Node) of the internal selected child Node
      6. {ParElementVar.childNodes[x].children[x].nodeValue;} => To Get the Value of the specific Element child (with certain index) of the internal selected child Node
  • {child Node}'s Type => {ParElementVar.childNodes[x].nodeType;}

    • This Method To get the Type of child Node, by numbers as following:

    • {1} : For {Element Child Node}
    • {2} : For {Attribute Child Node}
    • {3} : For {Text Child Node}
    • {8} : For {Comment Child Node}

JS DOM [Element Node Name Value Type] Importance & Benefits :

JS DOM [Elements Node Name Value Type] gives us the Following Features :

  • Providing more specified controing Tools of Element's Child Nodes's properties
  • Available for all Types of [Elements's Child Nodes] {Text, comment , Elements , Attributes , empty spaces } for advanced programing functions
  • Could be combined with other ChildNode Extentions Methods like (firstElementChild, lastElementChild, firstChild , lastChild , [specific Index] )
  • Could Include more exapanding methods which will provide more accurate Accessing Abilities
