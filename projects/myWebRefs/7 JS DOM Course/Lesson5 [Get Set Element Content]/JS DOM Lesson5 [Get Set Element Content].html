JS OOP [Get Set element Content] Concept:

=> JS HTML DOM [Get Set element Content]: They are OPPOSITE functions in programing generally & specially in dealing with [DOM] Elements.

=> JS HTML DOM [Get Set element Content]: The Main Methods are Usaually used in dealing with Get-Set fucnitons as following :

  • {document.element.innerHTML;} => Usually used to [get-set] (HTML) tags/Elements of to web Page [will be Strudied in this lesson]
  • {document.element.outerHTML;} => Usually used to [get-set] (HTML) tags/Elements of to web Page [will be study in later lesson]
  • {document.element.textContent;} => Usually used to [get-set] (Textual Content) in to web Page's Elements [will be study in later lesson]
  • {document.element.innerText;} => {Deprecated Method} Usually used to [get-set] (Texual Values) only in to web Page [will be Studied in this lesson]
  • {document.element.outerText;} => {Deprecated Method} Usually used to [get-set] (Texual Values) only in to web Page [will be Studied in this lesson]

=> JS HTML DOM [Differences between {.innerHTML;} & {.textContent;}]: The both are get-set Methods , but they have some defferent peorpties as following:

  1. [Function] => {.textContent}: is dealing with Textual content only , {.innerHTML} is dealing with both textual Content & HTML Elements Codes
  2. [Security] => {.textContent}: is much safer than {.innerHTML} due to innerHTML open space to hackers to re-Insert anew elements in page & and manipulate it's current Contents
  3. [Performance] => {.textContent}: is much effictive than {.innerHTML} due to innerHTML has much more data to load, but {textContent} only deal with String

=> To insert a conceptual HTML Codes inside the Page as Texual content by {.tectContent} you have to use the HTML special Symbols Codes such as :

  1. & gt; : for greater than Symbol [ > ]
  2. & lt; : for less than Symbol [ < ]
  3. & amp; : for And Symbol [ & ]
