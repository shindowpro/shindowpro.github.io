[Background New Features] Concept:

  • A) {Background New Features} main Concept :

    • In CSS3 there are much more features in properties to controling background image

    1. {background-size} :

      2. Default value of {background-size} is : [auto auto]

      This new feature to control {background image's size} , and can be set by following types of values:

      1. {background-size: #Width Value px height Value px;} :

        This type of value to control the image size by certain value of px

        The first value => Width & The Second value => height.

      2. background-size: width percentage % height percentage % :

        This one most popular type of value to be used

        The first value => Width & The Second value => height.

        This type of value has been fixed many of image sizing problems in CSS2 properties. such as {Justifing the inner backgournd image to fit accauratly it's parent contianer, or less }

      3. background-size: contain :

        This value to maintan the Image original (even if the parent scale chagned)

        This Value will repeat the image size if the parent scale is more/less the it's size

        This will not cut the original image size even the the parent scale changed

        This type of value is less popular to use due to image repeatation

      4. background-size: cover

        5. This is the most popular type of backgorund-size used, and is very usful in rersponsive Designing

        This value to maintan the Image scaling to be always fit it's parent container (even if it changed)

        This Value will repeat the image size if the parent scale is more/less the it's size

        This type of value will cut the image original size in case of the parent scale is smaller than the orginal image

        This type of value will stretch the image original size in case of the parent scale is bigger than the orginal image

        This type of value is less popular to use due to image repeatation

    2. {background-clip} :

      3. This new feature to control {background starting Edge} by setting by following types of values:

      Default value of {background-clip} is : [border-box]

      1. {background-clip: } : [border-box]

        This value will make the background edge start from it's parent contianer border

      2. {background-clip: } : [padding-box]

        This value will make the background edge start from it's parent contianer padding

      3. {background-clip: } : [content-box]

        This value will make the background edge start from it's parent contianer content

    3. {background-origin} :

      4. This new feature to control {background's image starting Edge} by setting by following types of values:

      Default value of {background-clip} is : [border-box]

      1. {background-clip: } : [border-box]

        This value will make the background edge start from it's parent contianer border

      2. {background-clip: } : [padding-box]

        This value will make the background edge start from it's parent contianer padding

      3. {background-clip: } : [content-box]

        This value will make the background edge start from it's parent contianer content
