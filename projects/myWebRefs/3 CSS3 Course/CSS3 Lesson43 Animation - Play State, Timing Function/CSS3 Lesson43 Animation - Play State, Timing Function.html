[CSS3 Animation - [Play-State, Timing Function] ] Concept:

{Animation- [Play-State || Timing-Function]} => Is Needing for using [prefixes] before the animation property and it's branshed properties

{Animation - [Play-State, Timing Function]}: These are the seventh and eighth child porperties of Animation syntax , with following details :

  1. { -webkit-animation-Play-State || animation-Play-State} =>

    It is the Play-Stating of the Animation

    [Default value] of Animation-Play-State : running

    Animation-Play-State => can be set by : {running} || {paused} : pausing property can be controled by using (javascript || J-Quarry)

  2. { -webkit-animation-Timing-Function || animation-Timing-Function} =>

    Which is the timing function mode (such as the transition funcitons)

    [Default value] : {ease}

    Can be assigned by one of following values :

    1. [ease] => To make the animating element start easy, then accelerating in the middle , then easy when finishing
    2. [ease-in] => To make the animating element start easy, then accelerating until finishing
    3. [ease-out] => To To make the animating element start then accelerating, finishing easy
    4. [ease-in-out] => To To make the animating element start easy, then accelerating in the middle , then easy when finishing [diffenret from te ease in more speedy in the middle]
    5. [linear] => To make the animating element with the same speed in whole the animation duration

[CSS3 / Animation - [Play-State || Timing-Function] ] Importance & Benefits:

=> [CSS3 / Animation - [Play-State || Timing-Function] ] : This Advanced Animation sub properties enabling the following features & imporatance :

  • [animation-Play-State] : Give a ability to set a certain timer for the animation by using [JS || J-quarry] to controling shifting the {play state} value
  • [animation-Timing-Function] : Give a ability to assign a certain Timing funciton mode to applay certain type of motion to the animation element {by several times}
