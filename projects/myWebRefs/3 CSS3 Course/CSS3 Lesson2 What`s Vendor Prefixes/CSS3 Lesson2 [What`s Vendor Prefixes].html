Vendor Prefixes [What`s Vendor Prefixes ] Concept:

=> Vendor Prefixes [Main concept]: They are certain starts of certain code CSS3 , which will provide some support for the next CSS3 code for certain Browsers.

=> Vendor Prefixes: They are (4) Vendors Prefixes :

  • -webkit- => Which are will allow supporting for following CSS3 code for following Browsers:
    • Google Chrome Browser
    • Safari Browser
    • Andriod Browser
    • IOS Browser
  • -moz- => Which are will allow supporting for Next CSS3 code for [Modzila Browser (fireFox)].
  • -o- => Which are will allow supporting for Next CSS3 code for [(Opera)] Browser
  • -ms- => Which are will allow supporting for Next CSS3 code for [(Microsft Internet Explorer)] Browser

=> Vendor Prefixes : Not All CSS3 is supporoting Vendor Prerfixes , and not all Browsers are the same supporting of the Same CSS3 property.

=> Vendor Prefixes [Future]: It wil be useless in future , due to they are used only for the old versions of Browsers, and all Compnies are Developing it's browers.

=> Vendor Prefixes [Compatablity]: Vendor Prefixes is compatible with all available internet Browsers, but not with all of CSS3 properties.

Vendor Prefixes [What`s Vendor Prefixes ] Importance & Benefits:

[Vendor Prefixes] gives us the Following Features :

  • It provides the Supporting for CSS3 code with browsers specialy with it's old Versions
  • It wil be useless in future , due to they are used only for the old versions of Browsers, and all Compnies are Developing it's browers.
  • Not all Browsers are the same in supporting the vendors prefixes.
