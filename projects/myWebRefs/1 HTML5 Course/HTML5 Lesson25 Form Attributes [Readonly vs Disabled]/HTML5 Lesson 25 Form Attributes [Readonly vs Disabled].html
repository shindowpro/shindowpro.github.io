Input Attribute [Readonly] Concept:

=> Input [Readonly Attribute] is one of the old HTML5 Form's Input Attributes, and it is commonly used by every programing language.

=> Input [Readonly Attribute] is to set the input feild as only able to be read, and NOT allowed to be Edited or Deleted.

=> Input [Readonly Attribute] can be allocated for more than allowed types of Input outside or inside the form with the page.

=> Input [Readonly Attribute] can NOT work with following Input types:

  • Hidden
  • Color
  • Submit
  • File
  • Checkbox
  • RadioButton
  • Range

=> Input [Readonly Attribute] must has the {Value} Attribute to guarantee that there are a contet to be displayed inside the input feild.

=> Input [Readonly Attribute] is allowing the input content to be sent to the server with other types of input in the form (Except Disabled Input feilds), inspite of its content is only can be readed and not able any Editing or deleting process by user.

Main & Features of Input Attribute [Readonly Attribute] Usages:

=> It mainly designed to prevent any ability of Editing or Deleting by user, and allowing each Copy process or just Reading it's contents.

=> It is a common way to display an Instant readable message to the user that can Not be deleted or Edited .

=> It allows to sending specific Content to the server with out any modification could made by user.

Main Differences between [Readonly] & [Disabled] Attributes:

Comparison between [Readonly] & [Disabled] Attributes
Description Readonly Disabled
Sending with Form input to Server Yes No
Copy Ability Yes No
Focus Yes No
Appearance Normal Gray
