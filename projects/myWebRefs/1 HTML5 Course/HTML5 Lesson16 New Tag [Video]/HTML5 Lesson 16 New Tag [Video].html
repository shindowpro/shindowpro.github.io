[Video Tag] Concept:

=> [Video Tag] is tag specified to deal with Video files on web Pages with advanced controls and tools in comparison with Html4 which were dealing with Video files less controls and features.

=> [Video Tag] has several properties inside:

  1. [Controls]: Which is allowing tools bar to the Video file coded.
  2. [Width = "#"]: This new feature is allowing to specify the [Width] of video frame, which was impossible in Html4.
  3. [Height = "#"]: This new feature is allowing to specify the [Height] of video frame, which was impossible in Html4.
  4. [Poster = ""]: Here where we can write the Image's Path to put it as a cover of Allocated video, And this Poster is temperorly coverd the video window unsless the user play the video.
  5. [Src = ""]: Here where we write the Video File Path, and this property can be written in the main < Video > Tag or it's partical tag < Source >
  6. [Autoplay]: It provides the (Auto play) feature to the allocated Video file once the website is loaded.
  7. [Loop]: It provides the Looping feature to the allocated Video file once it finished Automaticlay.
  8. [Mute]: It provides the ability to mute the Video file inspite of being played, and this can be a programable feature in Javascipt.
  9. [Preload = ""]: It provides the ability to control the download method of the Video file, and it offer (3) options :
    1. [Preload = "Auto"] : It is meaning to auto load the allocated Video file once the website is loaded.
    2. [Preload = "metadata"] : It is meaning to load the information of the allocated Video file once the website is loaded.
    3. [Preload = "none"] : It is meaning Not to load the allocated Video file while website is loaded unless the user click on play button form the control Video file.

=> [Source Tag] (< source >) is the first Partial Tag of [Video Tag], and it has 2 available properties:

  1. [src=""]: Which can be written as mentioned in Video Tag it self also, but it is better to write it in Source Tag , to have a chance to add many format type of Video file to gurantee that the browser will support one of file formats.
  2. [type = ""]: It specify the [ MIME ] type of the allocated Video file, and here some of mime type to common Video file extensions:
    1. (mp4) extension => (Video/mp4)
    2. (ogg) extension => (Video/ogg)
    3. (webm) extension => (Video/webm)
  3. [Media]: It is one of Unsupported property of [Source Tag] until (2014) to any Browser.

=> [Track Tag] (< track >) is the second Partial Tag of [Video Tag], which is meant by translation issues, and it might be more than one track tag for one video file and with different properties for each .And it has [4] available properties:

  1. [src = ""]: Here to write the [Translation file] [Web VTT] type with (.vtt) Extension.
  2. [kind = ""]: It specify the kind of translation allocated file within src pervious property, and it might be one of following [5] Kinds of track Translation Files:
    1. "Subtitle" => Which meant to specify the detected translation file as texual translator file kind.
    2. "Captions" => Which meant to specify the detected translation file as Audio Effections translator file kind.
    3. "Description" => Which meant to specify the detected translation file as texual description file kind for video contents.
    4. "Chapters" => Which meant to specify the detected translation file as dividing video to chapter file kind, and allowing to navigate through.
    5. "Metadata" => Which meant to specify the detected translation file as video track meta data description file kind.
  3. [Srclang = ""]: Here to write the [Iso Code] of language used in translation file, and each language has it's own Iso code, for example:
    • Arabic => [ar]
    • English => [en]
  4. [label = ""]: Here to write the Label or title of the this track.

Main & Features of [Video] Tag Usages:

=> It has solved many problems of older way in HTML4 with in controling Video file, by the [Video] tag & it's attributes and properties

=> It is very handy by using Javascript codes to manage it.

=> It is also can be formated by CSS codes.
