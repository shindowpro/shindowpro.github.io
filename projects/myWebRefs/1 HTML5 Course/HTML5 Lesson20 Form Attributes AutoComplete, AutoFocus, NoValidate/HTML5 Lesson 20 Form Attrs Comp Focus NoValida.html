Input Attribute [ < input   disabled > ]

Input Attribute [Disabled] Concept:

=> [Disabled Attribute] is one of the new important properties of Input in HTML5.

=> [Disabled Attribute] is to Cancel any Inserting / Editing Process in a specific input field .

Main & Features of Input [Disabled Attribute] Usages:

=> It mainly designed to cancel any inserting process or Editing by user with out pre- programing efforts.

=> It can be programatically used in Events to as an actions or reactions to make change by page or any of its element/s using Javascript codes in front end.

(Form / Form's Input) New Attribute [Autocoplete] Concept:

=> [Autocomplete Attribute] is one of the new (Form / Form's Input) Attribute in HTML5.

=> [Autocomplete Attribute] is property allowing to show the user the previous Entries inserted in the input field or the Whole Form when it is activated "on" which is the defaulted value, but can be canceled or deactivated by giving it "off" value within the desired input field Tag's Attributes.

=> [Autocomplete Attribute] is mainly designed to deactivate input field/form to acheive secured field Input entries within Form by canceling this property, or activate it by default.

Main & Features of (Form / Form's Input) [Autocomplete] Attribute Usages:

=> It mainly designed to secure specific input fields contents by canceling or deactivating this property for the desired input field.

=> It can be programatically used in Events to as an actions or reactions to make change by page or any of its element/s using Javascript codes in front end.

Form => Input New Attribute [ < input   autofocus="on" / "off" > ]

Form Input New Attribute [Autofocus] Concept:

=> [Autofocus Attribute] is one of the new Form Input field Attributes in HTML5.

=> [Autofocus Attribute] is a property to make the cursor inside a certain desired input field within the form once the page is loaded.

Main & Features of Form Input [Autofocus Attribute] Usages:

=> It can be be programatically used in Events to as an actions or reactions to change the Value of certain input or element/s of page using Javascript codes.

=> It provide an easy method to set the cursor inside a certain input field.

Example:

Setting the cursor inside a specific input field

The Cursor will be inside the following input field once the page is loaded(AutoFocus):

   <h3> The Cursor will be inside the following input field once the page is loaded: </h3>

     <input   type = "submit"  value = "send" >
   </form>
</section>
Form New Attribute [ < form   novalidate > ]

Form New Attribute [NoValidate] Concept:

=> [NoValidate Attribute] is one of the new Form (itself) Attributes in HTML5.

=> [NoValidate Attribute] is to cancel the auto Validating property of the input field Contents, by only add it within The Form Attributes.

=> [NoValidate Attribute] is mainly made to cancel Validation Process by certain conditions (by trusting certain users accounts)

=> The Default Validation Property of any input field in any Form is Activated Validation, but NoValidate isDeactivating this property for the input fields in allocated Form.

Main & Features of Form Attribute [NoValidate] Usages:

=> It mainly designed to remove Validating property over a certain person or account in certain Form.

=> It can be programatically used in Events to as an actions or reactions to make change by page or any of its element/s using Javascript codes in front end.
