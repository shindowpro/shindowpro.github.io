JS OOP [ Introduction - What is OOP ] Concept:

=> JS OOP [ Introduction - What is OOP] : This Course Will Present advanced knowledge of JS
spcially and all P.languages which happening behind the scence of Front End Interface.

=> JS OOP [OOP Terminology] : [OOP] is refers to (Object Oriented Programming).

=> JS OOP [ Introduction - What is OOP] : Is a Paradigm or Style of Coding scripting which is
using Objects Concept to desing programes and applications.

=> JS OOP [ Introduction - What is OOP] : Is NOT a programing language or programatic Tool ,
but it is a programatic Style or Method.

=> JS OOP [ Introduction - What is OOP] : Is an old programatic Style or Method which is
existed since 2960.

=> JS OOP [ Introduction - What is OOP] : Not all Programming languages has OOP , but there
are has this style and other are not, and the following has OOP :

  • VB.net
  • Python
  • C#
  • C++
  • Java
  • JavaScript
  • PHP

=> JS OOP [ Object ] Concept : It is a programtic Package that has specific [Methods (Fucntions)]
and [properties], that work together to provide certain programatic performance or
Function(s)

JS OOP [ Introduction - What is OOP] public Comparison between (Procedural Programming) vs (OOP) :

  1. [ Procedural Programming ] :

    2. => A) (Application Elements) : Program or Application is Divided into :

    • Variables
    • Functions

    => B) (Overloading) : Overloading feature IS NOT Possible and Available

    => C) (Hiding Data) : Hiding Data is NOT Available and Possible

    => D) (Data Community) : Data is Separated

  2. [ OOP ] :

    3. => A) (Application Elements) : Program or Application is Divided into :

    • Objects
    • Methods

    => B) (Overloading) : Overloading feature is Possible and Available

    => C) (Hiding Data) : Hiding Data is Available and Possible (By Encapsulation)

    => D) (Data Community) : Data is one Community or Location
    (Class : Objects >= Method(s) >= Properties)

JS OOP [ Introduction - What is OOP] Advantages & Benefits:

=> JS OOP [ OOP Advantages ]: In general has following Features :

  • It is the best and Ideal way or Method to deal with large scale of Data in Complex Software Arcetitures
  • The Ability to hide partial part of Objects Codes by using [Encapsulation] Technique which is available in OOP , to preveny missing with the Codes
  • The Ability to create Re-usable Objects to reuse them in other locations inside the same appliation, by using [Inheritance] Technique available only by OOP, to allow re-useing other Objects Properties within other Objects Method
