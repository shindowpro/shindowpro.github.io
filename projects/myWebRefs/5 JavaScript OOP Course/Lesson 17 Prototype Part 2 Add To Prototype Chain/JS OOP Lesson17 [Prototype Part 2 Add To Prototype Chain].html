JS OOP [Add To Prototype Chain ] Concept:

=> JS OOP [Add To Prototype Chain ]: This lessson will provide the Method of adding methods and Properites to the Constructor function's Protptype to be accesed by the constructor function and all it's Extracted functions .

=> JS OOP [Add To Prototype Chain ]: The Prototype Will be Generated autumatically once the constructor functions (of any type) is being created.

=> JS OOP [Add To Prototype Chain ]: The Prototype is also being created to any Variable or Object (to it's Built in Constructor Function).

=> JS OOP [Add To Prototype Chain ]: We can Add new Methods and Properties to the Prototype of the Constructor Function.

=> JS OOP [Add To Prototype Chain ]: The Prototype can Access the new added Methods and Properties (as it in the Extracted Objects) .

=> JS OOP [Add To Prototype Chain ]: The (Prototype / Extracted Object / Constructor Function) is reading Any Defined Property or Method inside the Constructor Function upon certain Mechanism [Chain] as following:

  1. [ Main Constructor Function ] =<

    2. Reading propteies & Methods of the Main Constructor Function is the First thing the Prototype/Object is reading, if it has been found , it will be access

  2. [ Main Constructor Function's Prototype] =<

    3. Reading the Propeties and Methods of First Prototype in the Constructor Function, if the Prototype/Object has been found , it will be access

  3. [Built in Constructor Function Prototype] =<

    4. Reading the Propeties and Methods of Next Prototype in the Constructor Function, if the Prototype/Object has been found , it will be access

=> JS OOP [Add To Prototype Chain ]: If any Method/Property has been added to the Prototype of the Built in Constructor Function , it will be accessed by any new same Data type Variable / Object will be defined (as it's the Built in Constructor Function)

JS OOP [Add To Prototype Chain ] Importance & Benefits:

[Add To Prototype Chain ] gives us the Following Features :

  • More advanced tools to extract objects from Funcitons Wether it's type was [Constructor Function , Normal Function, Specified Data type Built In Constructor function ].
  • Ability to inherate all certain Function's properites and Methods = < as an Automated generated Object (with all Inhratance abitlies).
  • Limitless number of automated Generated Chained Prototypes which is being created to any Function / Variable's Built in Constructor Function (with all the constructor Function Properites and mehtods).
  • The Ability to Full Controling of the Prototypes by :
    • Adding new Properites and Methods
    • Deleting new Properites and Methods
    • Accessing to any Property or Method (In the Main Constructor Function - in one of it's Prototype )

JS OOP [Add To Prototype Chain ] Syntax & Methods:

=> JS OOP Defining Object by [Add To Prototype Chain ] Conceptual Examples as following :

// A) Main Syntax adding new (Property or Method) to the Prototype :

ConstructorFunctionName.prototype.NewMethod/PropertyName = "value" / function() {}
//===========================================================

//===========================================================

// B) Conceptual Example Syntax of adding new (Property or Method) to the Prototype :

// (a) Defining Conceptual Main Constructor function :
function ConstFuncName(par1) {
this.par1= par1;
this.Method1 = () => {
return 'par1';
}
}

//===========================================================

// (b) Adding new [Property] to a Prototype of a Constructor Function :

ConstructorFunctionName.prototype.NewPropName = 'Value';
//===========================================================

// (c) Adding new [Method] to a Prototype of a Constructor Function :

ConstructorFunctionName.prototype.NewMethName = function() {
return 'value ${this.par1}';
};
//===========================================================

// (d) Accessing & Printing new [Method] & [Property] added to Constructor Function's Prototype :

// (*) Accessing through the main Constructor Function :

// d1) Accessing the New Method :
console.log(ConstructorFunctionName.NewMethName());
//===========================================================

// d2) Accessing the New Property :
console.log(ConstructorFunctionName.NewPropName);
//===========================================================
//===========================================================

// (*) Accessing through the main Constructor Function's Prototype :

// d1) Accessing the New Method :
console.log(ConstructorFunctionName.prototype.NewMethName());
//===========================================================

// d2) Accessing the New Property :
console.log(ConstructorFunctionName.prototype.NewPropName);
//===========================================================

//===========================================================

//===========================================================

// (e) Defining Extracted Object form the main Conceptual Main Constructor function + Accessing the new Properties and Methods through it's Prototype:

//(e1) Defining the new Extracted Object :
const newObj = new ConstFuncName();
//===========================================================

//(e2) Accessing the new Properties and Methods through the new Extracted Object:
//(*) Accessing the new Properties and Methods through the Main Extracted Object :
console.log(newObj.NewPropName);
console.log(newObj.NewMethName);
//===========================================================

//(*) Accessing the new Properties and Methods through the Prototype of Extracted Object :
console.log(newObj.prototype.NewPropName);
console.log(newObj.prototype.NewMethName);
//===========================================================
