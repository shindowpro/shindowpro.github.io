JS OOP [Prototype Chain ] Concept:

=> JS OOP [Prototype Chain ]: This lessson will provide Extra Advanced concept of the prototype & Constructor function .

=> JS OOP [Prototype Chain ]: Each [Constructor Function] has a [Prototype].

=> JS OOP [Prototype Chain ]: Each [Object] has a [Prototype] (becasuse the Object has a Constructor Function ) .

=> JS OOP [Prototype Chain ]: Each [Varaible] has a [Prototype] (becasuse any Varialble has a Built in Constructor Function ) .

=> JS OOP [Prototype Chain ]: Prototype is an [Object] , which is displaying all it's (Object/Variable/Function)'s Constructor Funciton as one of It's Properites and Values .

=> JS OOP [Prototype Chain ]: Prototype's Chain is about the Sequence of created Prototypes (because Constructor Function has a prototype & Prototype (Object) has a Prototype also)

=> JS OOP [Prototype Chain ]: Prototype's Chain ends with [Prototype of Object Built in Constructor Function (Prototype.Object function)]

=> JS OOP [Prototype Chain ]: Each Defiend Variable is an Extracted Object from it's [Built in Consturctor Function] of it's Data type, so it has [Prototype]

=> JS OOP [Prototype Chain ]: The [Prototype] of the {Prototype} is a built in Object Constructor Function.

JS OOP [Prototype Chain ] Importance & Benefits:

[Prototype Chain ] gives us the Following Features :

  • Great understanding of OOP of Prototype gives great abilities for more advanced programing features.
  • More advanced tools to extract objects from Funcitons Wether it's type was [Constructor Function , Normal Function, Specified Data type Built In Constructor function ].
  • Limitless number of automated Generated Chained Prototypes which is being created to any Function / Variable's Built in Constructor Function (with all the constructor Function Properites and mehtods).
  • The Ability to Full Controling of the Prototypes by :
    • Adding new Properites and Methods
    • Deleting new Properites and Methods
    • Accessing to any Property or Method (In the Main Constructor Function - in one of it's Prototype )
    • Full Access with unlimited time to the added Method(s) to a Built in Prototype of A constructor Function

[Prototype Chain ] Operational Proccess of Extracting new Object from Consturctor Functions as following :

  1. Creating a new Empty Object
  2. Assinging {this} Contax to the [new Empty Object] =< this ={};
  3. Assinging [_proto_] contax of {this}(which is belongs to empty Ojbect) to [_proto_] contax of {Constructor function} :
    this._proto_ = ConstructorFunction._proto_;
  4. Linking between _proto_ of new extrated Object with _proto_ of Constructor Function
    User._proto_ = user1._proto_ ;

JS OOP [Prototype Chain ] Syntax & Methods:

=> JS OOP Defining Object by [Prototype Chain ] Conceptual Examples as following :

// A) Defining a Default Constructro Function :

function User(name) {
this.name = name;
}
//===========================================================

//===========================================================

// B) Extracting a new Object from constructor Function [Instace & Not Instance Object ]:

// 1] Creating an Instance Object using [new keyword]:
let user1 = new User("TextValue");

// 2] Creating same Object without using [new keyword]:
let user2 = User("TextValue");

//--------------------------------------------------------------------------------------------------

// C) Accessing to Prototype of new Extracted Object :

console.log(user1.prototype);

//--------------------------------------------------------------------------------------------------

// D) Accessing to Prototype of Main Constructor Function :

console.log(User.prototype);

//--------------------------------------------------------------------------------------------------

// E) Checking if the Defined object by using [new Keyword] or not :
// First Method [Checking by instanceof keyword] :
if (!(this instanceof User)) {
throw new Error("The Extracted Object is not by (new Keyword)");
console.log("The Extracted Object is not by (new Keyword)");
}
//===========================================================
