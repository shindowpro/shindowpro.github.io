JS OOP [JavaScript Accessors Getters & Setters ] Concept:

=> JS OOP [JavaScript Accessors Getters & Setters ]: It is one of Powerful points of programing in any P.Language.

=> JS OOP [JavaScript Accessors Getters & Setters ]: It will provide the advanced Control for Accessibility to [Private Methods & Properties] of the Class or Constructor function .

=> JS OOP [JavaScript Accessors Getters & Setters ]: JavaScript is not Supproting all Features of Getteres And Setters Accesors, but it all Supproted in All other P.Languages.

=> JS OOP [JavaScript Accessors Getters & Setters ]: It will give the Feature of simple syntax comparison with the defualt Method.

=> [get] Accessor => Advanced way to Get certain value from Certain Method with certain Processing, in secured Procedures .

=> [get] Accessor => It can be added before the simple method (of Geting certain Value Method) of the Class or Consturctor function.

=> [get] Function => Getting Function can be Accessed by two ways:

  1. [Defualt way] => By the Default Method Of Class or Constructor Function {which has (result) keyword}
  2. [GET Accessor] => By using {get} before the Normal Method (of Geting function)

=> [set] Accessor => Advanced way to Set certain value/change in Certian Variable or Parameter, within Certain Method by certian Processing, in secured Procedures .

=> [set] Accessor => It can be added before the simple method (of Setting certain Value/Change Method) of the Class or Consturctor function.

=> [set] Function => Getting Function can be Accessed by two ways:

  1. [Defualt way] => By the Default Method Of Class or Constructor Function {which has (result) keyword}
  2. [SET Accessor] => By using {set} before the Normal Method (of Setting function)

JS OOP [JavaScript Accessors Getters & Setters ] Importance & Benefits:

[JavaScript Accessors Getters & Setters ] gives us the Following Features :

  • It will give the Feature of simpler syntax comparison with the default Method
  • It gives secured Access to the Private Method in Class or Constructor Function in all other P.languages.
  • {get} Accessor It more advanced and Easy way to [get] specfic Value or Procedures [specially in complicated Methods] in advaced application or web Pages , through seured Mechanism
  • {set} Accessor It more advanced and Easy way to [set] specfic Value or Procedures [specially in complicated Methods] in advaced application or web Pages , through seured Mechanism

JS OOP [JavaScript Accessors Getters & Setters ] Syntax & Methods:

=> JS OOP Defining-Declaring [Class] Conceptual Syntax as following :

// A) JS OOP Defining-Declaring [Class] Conceptual Syntax as following :

class ClassName {
constructor(Par1, Par2) {
this.par1 = par1;
this.par2 = par2;
}

// A 1) Creating/Defining [Getting Methods (Par1 + Par2)]:
a) Normal get Method:
showAll() {
return 'The first Value of Parameter1 is ${Par1}\n The Second Value of Parameter2 is ${Par2};
}

b) {get} Accessor Method:
get showAll() {
return 'The first Value of Parameter1 is ${Par1}\n The Second Value of Parameter2 is ${Par2};
}


//===========================================================

//===========================================================

// A 2) Creating/Defining [Setting Methods (Par1 + Par2)]:
a) Normal set Method:
changeAll(newPar1, newPar2) {
this.par1 = newPar1;
this.par2 = newPar2;
}

b) {set} Accessor Method:
set changeAll(newPar1, newPar2) {
this.par1 = newPar1;
this.par2 = newPar2;
}


}


//===========================================================

//===========================================================

// B) Extracting an Object from Class:
let nObj = new ClassName(par1, par2); //===========================================================

// C) Executing [Getting Methods (Par1 + Par2)]:
a) Normal get Method:
nObj.showAll();

b) {get} Accessor Method:
nObj.showAll;

//===========================================================

// D) Executing [Setting Methods (Par1 + Par2) {for both Normal + {set} Accessor}]:
{set} Method + Normal Method:
nObj.changeAll("newPar1", "newPar2");

//===========================================================

//===========================================================
