JS OOP [Class Static Properites and Methods ] Concept:

=> JS OOP [Class Static Properites and Methods ]: [Static Property & Method] is defined Properites and Methods that not related to any of Extracted objects that will be Created from this Classs.

=> JS OOP [Static Properites ]: They are Properties that are not Related of any Extracted objects will be Extracted from the Class.

=> JS OOP [Static Properites ]: It can be Accessed only by Class itself , and Not by any of Extracted Object(s).

=> JS OOP [Static Method ]: They are Methods that are not Related of any Extracted objects will be Extracted from the Class.

=> JS OOP [Static Method ]: It can be Accessed only by Class itself , and Not by any of Extracted Object(s).

=> JS OOP [Class Static Properites and Methods ]: This lessson will provide advanced concepts & Examples of Static Propeties and Methods + Extra Differences between [Constructor Function and Class].

JS OOP [Class Static Properites and Methods ] Importance & Benefits:

[Class Static Properites and Methods ] gives us the Following Features :

  • Great understanding of OOP of Class Static Properites and Methods gives great abilities for more advanced programing features.
  • More advanced Accessing tools to different scopes of Class, and gives Private Sopes of Accessibility for only Class without any of Extracted Objects .
  • Accessibility Features to Access to [Static Variable] Dircecty through [Static Method] & Indirectly through [Normal Method by using Class name itself] .
  • The Ability for Extra Controling of Class through Static Properites and Methods, with out Effecting on any of Extracted Object(s)

[Class] is Conceptually differing from [Constructor Function] in following Aspects :

  1. In [Class] any Extracted object(s) must be Declared After Decleration of [Class] it-self, which the Both is allowed in [Constructor Function ]
  2. In [Class] Properties and Parameters is being declaring by {constructor} syntax, which is a part of [Class] main Syntax.
  3. In [Class] Method(s) can be Extracted from {Constructor} and be placed in separeted syntax, but inside the main [Class].
  4. Error throw Message {of Not using [New] key wordf in Extacting new Object} in [Class] will be showen by Default, but in Constructor Function needs to Be written in the script

=>Following are Differences between [Static and Normal Methods & Properties] :

  1. [ Using {this.} ] => :
    • [Outside constructor ] => Defining Properties & Methods in Class outside {constructor} syntax we Dont need to use {this.} for each (Static & Normal) Methods & Properties.
    • [Inside constructor ] => Defining Properties & Methods in Class inside {constructor} syntax
      • [ Normal Methods & Properties] => We Must need to use {this.}
      • [ Static Methods & Properties] => We Must need to use {Class's Name.} istead of this.
  2. [ using {=} keyword ] :
    • [Static Methods & Properties] => We Must use [=] in the Syntax always {although it is out side the (Constructor)}
    • [Normal Methods & Properties] => There are (2) cases {inside & outside (constructor) keyword}:
      • [Inside constructor keyword] => We Must use [=] & [this.] keywords in the Syntax
      • [Outside constructor keyword] => We Must NOT use [=] in the Syntax for [Normal Methods & Properties only]
  3. Accessability to Static properites of Class inside methods (Normal & Static) :
    • [Inside Static] Method => We can use just [this.] Operator to Access Directly the Static Variable , {because [this] inside the Static access to [Class] itself}
    • [Inside Normal] Method => We Must use [Class Name] to Access to Static Variable/Method, and [this.] Opertator can not be used {because it access only of Parameters inside [constructor] Syntax}

// A) JS OOP Defining-Declaring [Class] Conceptual Syntax as following :
